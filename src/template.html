📖 How It Works

Write a graph traversal algorithm. The visualizer will animate your explored edges (blue) then draw the shortest path (red). Pick start & end on the map first.

📦 Data You Receive

graph.nodes { "12345": { lat: 3.05, lng: 101.6 }, ... }
graph.edges { "12345": [{ to: "67890", length: 142.5, coords: [[lat,lng], ...] }] }
startId / endId Node ID strings matching keys in graph.nodes

✅ What To Return

LanguageReturn Format
JavaScript return { exploredEdges: [[[lat,lng],...], ...], path: [[lat,lng], ...] }
Python def run_algorithm(graph, start_id, end_id):
  return {"explored_edges": [...], "path": [...]}
C++ cout << json{{"explored", [["u","v"],...]}, {"path", ["n1","n2",...]}}

💡 Tips

  • exploredEdges — array of coordinate arrays (one per edge visited)
  • path — flat array of [lat, lng] pairs forming the route
  • Use nb.length as edge weight (meters)
  • Use nb.coords to get the road geometry for each edge
  • C++ uses Wandbox — keep subgraph small for fast compile
